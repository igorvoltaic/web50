Registry

USER

added docker.registry.htb to hosts and starget gobuster against https on port 443

fond a path to /v2

and read the HTTP response header which said: Docker-Distribution-Api-Version registry/2.0

it meant that this box is running docker and turned out /v2 stands for API

to get image name GET this path /v2/_catalog

to make docker pull working 1st we need to setup cert which is done with this command:

openssl s_client -showcerts -connect docker.registry.htb:443 < /dev/null | sed \
    -ne '/-BEGIN CERTIFICATE-/,/-END CERTIFICATE-/p' > /etc/docker/certs.d/docker.registry.htb/ca.crt

Login creds were simple: admin:admin

docker login -u admin -p admin docker.registry.htb:443/bolt-image

Now i can pull the container:

docker pull docker.registry.htb:443/bolt-image

Let's run it and see what's inside.

docker run -it 601499e98a60i

Now I can grab id_rsa for bolt user but it has a pass phrase

Also I have found out that /var/www/html/sync.sh uses ssh but runs without any pass phrase

I checked .viminfo and found the last edited file and that's the file with the pass phrase..

restic init --repo ./tmp
